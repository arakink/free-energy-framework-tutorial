# exercise2_phi_simulation.py の数理的な解説

## このファイルでやっていること

このファイルは、練習問題1のように事後分布 `p(v|u)` 全体を求めるのではなく、

「観測 `u` が与えられたときに、最ももっともらしいサイズはどこか」

を表す代表値 `φ` を、時間発展によって求めています。

つまり、ここでやっているのは

- 分布全体を描くこと

ではなく、

- 分布の山の頂上に対応する代表値を探すこと

です。

## 基本の考え方

練習問題1では、求めたい量は事後分布

`p(v|u)`

でした。

練習問題2では、この分布全体を持つ代わりに、

`φ`

という1つの変数を使って

「今もっともらしいと考えているサイズの推定値」

を表します。

そして、この `φ` を少しずつ更新して、最終的に

`p(v|u)` が最大になる場所

へ近づけます。

## ベイズ推定から出発する

出発点はベイズの定理です。

`p(v|u) = p(v)p(u|v) / p(u)`

ここで観測 `u` はすでに与えられているので、`φ` を変えたときに変わるのは主に

- `p(φ)`
- `p(u|φ)`

です。

分母 `p(u)` は `φ` に依存しないため、最大値を探すときには無視できます。

したがって、

`p(φ|u)` を最大化する

ことは、

`p(φ)p(u|φ)` を最大化する

ことと同じです。

## なぜ対数を取るのか

論文では次の量を定義します。

`F = ln p(φ) + ln p(u|φ)`

これは

`F = ln ( p(φ)p(u|φ) )`

と同じです。

対数を取る理由は2つあります。

- 積が和になるので扱いやすい
- 対数は単調増加なので、最大になる `φ` が変わらない

したがって、`p(φ)p(u|φ)` を直接最大化する代わりに、`F` を最大化してよいことになります。

## F の意味

`F` は、

- `φ` がどれだけ事前知識に合っているか
- `φ` がどれだけ観測 `u` をうまく説明できるか

を合わせた評価関数です。

この例では、正規分布を代入すると、`F` は本質的に

- `φ` が事前平均 `vp` からどれだけずれているか
- `g(φ)` が観測 `u` からどれだけずれているか

の二乗誤差に基づく式になります。

そのため、`F` を大きくすることは、

- 事前からのずれを小さくする
- 観測とのずれを小さくする

ことに対応します。

## 式(8)の意味

次に論文では

`∂F/∂φ`

を計算します。

これは、

「今の `φ` をどちら向きに動かせば `F` が大きくなるか」

を表します。

この例では

`∂F/∂φ = (vp - φ)/Σp + ((u - g(φ))/Σu) g'(φ)`

です。

この式は2つの項に分かれています。

- `(vp - φ)/Σp`
  - `φ` を事前平均 `vp` の方向へ引っ張る
- `((u - g(φ))/Σu) g'(φ)`
  - `φ` が観測 `u` をよりよく説明する方向へ動く

つまり、`φ` は

- 事前知識
- 観測による誤差修正

の両方の影響を受けながら動きます。

## 式(9)の意味

論文では次に

`dφ/dt = ∂F/∂φ`

という更新則を置きます。

これは、

「`F` が増える方向へ `φ` を時間とともに動かす」

という勾配上昇法です。

意味としては次の通りです。

- `∂F/∂φ > 0`
  - `φ` を増やすと `F` が増える
- `∂F/∂φ < 0`
  - `φ` を減らすと `F` が増える
- `∂F/∂φ = 0`
  - その場では更新が止まる

したがって、更新を繰り返すと `φ` は `F` の高い場所へ近づいていきます。

ここで出てくる時間 `t` は、物理的な時間そのものというより

- 推定の進行
- 更新の過程
- 神経活動の緩和過程

を表すために導入されたものです。

## コードで何をしているか

このファイルの `dphi` 関数は、式(8)に対応しています。

`g(φ)=φ^2` なので `g'(φ)=2φ` となり、更新量は

`(vp - φ)/Σp + ((u - φ^2)/Σu) 2φ`

となります。

そして `simulate_phi` では、これを離散時間で

`φ_new = φ_old + dt * dφ`

の形で繰り返しています。

これは微分方程式

`dφ/dt = ∂F/∂φ`

のオイラー近似です。

## 練習問題1との違い

練習問題1と練習問題2の違いは次の通りです。

- 練習問題1
  - `p(v|u)` 全体を計算する
- 練習問題2
  - `p(v|u)` の最大点だけを近似的に探す

したがって、練習問題2は

「分布全体を表すのは大変なので、代表値だけを動的に求める」

という近似推論になっています。

## グラフの意味

このファイルが描くグラフは

- 横軸: 時間
- 縦軸: `φ`

です。

つまり、

「推定値 `φ` が時間とともにどう変化し、どの値へ収束していくか」

を示しています。

このグラフは事後分布そのものではなく、

`p(v|u)` の頂上を探す過程

を可視化したものです。

## 読み方のポイント

このファイルを理解するには、次の順番で考えると整理しやすいです。

1. まず `p(v|u)` 全体を求める代わりに、代表値 `φ` だけを持つ
2. `F = ln p(φ) + ln p(u|φ)` を定義する
3. `∂F/∂φ` で、どちら向きに動けばよいかを調べる
4. `dφ/dt = ∂F/∂φ` に従って `φ` を更新する
5. 最終的に `φ` がもっともらしい値に収束する

この流れで見ると、練習問題2は

「ベイズ推定を、時間発展する1個の内部状態で近似している」

ものだと理解できます。
