# exercise3_factor_graph_simulation.py の数理的な解説

## このファイルでやっていること

このファイルは、練習問題2で1本の更新式として書いていた推定過程を、`phi`、事前誤差 `epsilon_p`、感覚誤差 `epsilon_u` という複数の状態に分けて表しています。

目的は練習問題2と同じです。

`p(v|u)` が最大になるような、もっともらしいサイズの推定値を探します。

違いは、誤差をその場で直接計算するのではなく、誤差そのものを別ノードの活動として持つ点です。

## 練習問題2との関係

練習問題2では、`phi` の更新は次の形でした。

`dphi/dt = (vp - phi)/Sigma_p + ((u - g(phi))/Sigma_u) g'(phi)`

これは、事前誤差と感覚誤差をその場で計算して、すぐに `phi` の更新に使う式です。

練習問題3では、この2つの誤差を新しい変数として分けます。

`epsilon_p = (phi - vp) / Sigma_p`

`epsilon_u = (u - g(phi)) / Sigma_u`

このように置くと、`phi` の更新は次のように書けます。

`dphi/dt = -epsilon_p + epsilon_u g'(phi)`

つまり、`phi` は事前誤差と感覚誤差から入力を受けて更新されます。

## 予測誤差ノードの意味

練習問題3で重要なのは、`epsilon_p` と `epsilon_u` を単なる計算結果ではなく、時間発展する状態として扱うことです。

事前誤差 `epsilon_p` は、現在の推定値 `phi` が事前平均 `vp` からどれだけずれているかを表します。

感覚誤差 `epsilon_u` は、実際の観測 `u` と、現在の推定値から予測される観測 `g(phi)` がどれだけずれているかを表します。

この2つが、予測符号化でいう「予測誤差」に対応します。

## 式(13)の意味

式(13)は、事前誤差ノードのダイナミクスです。

`depsilon_p/dt = phi - vp - Sigma_p epsilon_p`

この式は、`epsilon_p` が最終的に

`epsilon_p = (phi - vp) / Sigma_p`

へ近づくように作られています。

平衡状態では `depsilon_p/dt = 0` なので、

`0 = phi - vp - Sigma_p epsilon_p`

となります。これを `epsilon_p` について解くと、

`epsilon_p = (phi - vp) / Sigma_p`

になります。

つまり式(13)は、事前平均との差を表す誤差ノードが、正しい誤差値へ時間をかけて収束するための式です。

## 式(14)の意味

式(14)は、感覚誤差ノードのダイナミクスです。

`depsilon_u/dt = u - g(phi) - Sigma_u epsilon_u`

これも同じ構造です。

平衡状態では `depsilon_u/dt = 0` なので、

`0 = u - g(phi) - Sigma_u epsilon_u`

となります。これを `epsilon_u` について解くと、

`epsilon_u = (u - g(phi)) / Sigma_u`

になります。

つまり式(14)は、観測と予測のずれを表す誤差ノードが、正しい感覚誤差へ時間をかけて収束するための式です。

## 遅れて追随するという意味

練習問題2では、誤差は今の `phi` から即座に計算されました。

練習問題3では、`epsilon_p` と `epsilon_u` は微分方程式に従って少しずつ変化します。

そのため、`phi` が変わった直後には、誤差ノードはまだ新しい `phi` に完全には対応していません。コードの離散更新で見ると、`phi` の更新には一つ前のステップで保持されていた `epsilon_p` と `epsilon_u` が使われます。

これが「誤差が遅れて追随する」という意味です。

この遅れがあるため、練習問題3の `phi` の軌跡は練習問題2と完全には一致しません。複数のノードが相互に影響するので、収束が遅くなったり、振動的になったりします。

## コードとの対応

コードでは、`error_p` が `epsilon_p`、`error_u` が `epsilon_u` に対応します。

`phi` の更新:

```python
phi = phi + dt * (-error_p + error_u * (2.0 * phi))
```

これは

`dphi/dt = -epsilon_p + epsilon_u g'(phi)`

に対応します。ここでは `g(phi)=phi^2` なので、`g'(phi)=2phi` です。

事前誤差の更新:

```python
error_p = error_p + dt * (phi - v_p - sigma_p * error_p)
```

これは式(13)に対応します。

感覚誤差の更新:

```python
error_u = error_u + dt * (u - phi**2 - sigma_u * error_u)
```

これは式(14)に対応します。

## グラフの意味

このファイルが描くグラフは、次の3つの時間変化です。

- `phi`
- `epsilon_p`
- `epsilon_u`

練習問題2のグラフは `phi` だけでした。

練習問題3では、`phi` がどのように収束するかに加えて、事前誤差と感覚誤差がどのように立ち上がり、変化し、落ち着いていくかも見ます。

## 読み方のポイント

練習問題3は、練習問題2の更新式をより神経回路らしく分解したものです。

練習問題2:

`phi` が、その場で計算された誤差を使って直接更新される。

練習問題3:

`phi`、`epsilon_p`、`epsilon_u` が別々のノードとして存在し、それぞれ時間発展しながら相互作用する。

この違いにより、最終的に似た値へ向かっていても、途中の軌跡は異なります。
