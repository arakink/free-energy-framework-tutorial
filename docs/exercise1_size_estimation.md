# exercise1_size_estimation.py の数理的な解説

## このファイルでやっていること

このファイルは、観測された光強度 `u` から、物体サイズ `v` がどれくらいありそうかをベイズ推定で求めています。

求めたい量は、事後分布

`p(v|u)`

です。これは

「観測 `u` を見たあとで、サイズ `v` がどれくらいもっともらしいか」

を表します。

## 問題設定

この例では、次のように考えます。

- `v`
  - 推定したい隠れた変数。物体サイズ
- `u`
  - 実際に観測された値。光強度
- `g(v)`
  - サイズ `v` から期待される光強度を与える関数

ここでは

`g(v) = v^2`

を使います。つまり、サイズが大きいほど観測される光強度が大きくなると仮定しています。

## 観測モデルの意味

まず、サイズ `v` が決まったときに観測 `u` がどう生じるかを

`p(u|v) = f(u; g(v), Σu)`

で表します。

これは

「サイズが `v` なら、観測 `u` は平均 `g(v)` のまわりにノイズ付きで現れる」

という意味です。

ここで重要なのは、`p(u|v)` は `u` と `v` の関係を記述しているという点です。

- `v` を固定して `u` を動かして見ると、平均 `g(v)` を中心とする正規分布になります
- `u` を固定して `v` を動かして見ると、観測 `u` をよく説明する `v` のところで値が大きくなります

今回のコードでは後者を使っています。つまり、`u=2` を固定して、

「どの `v` がこの観測をよく説明するか」

を見ています。

## 事前分布の意味

観測を見る前から、サイズについての先入観があると考えます。それが

`p(v)`

です。

この例では、サイズは平均 `3` のまわりに出やすい、という事前分布を置いています。

したがって、

- 観測だけを見れば `u=2` を説明するサイズが有力
- 事前知識だけを見れば `v=3` 付近が有力

という2つの情報が存在します。

## ベイズの定理の意味

この2つを結びつけるのがベイズの定理です。

`p(v|u) = p(v) p(u|v) / p(u)`

意味としては、

「観測後の `v` のもっともらしさ」
=
「観測前の `v` のもっともらしさ」
×
「その `v` が今回の観測をどれくらい説明できるか」
÷
「全体のつじつま合わせ」

です。

各項の役割は次の通りです。

- `p(v)`
  - 事前分布
- `p(u|v)`
  - 尤度
- `p(v|u)`
  - 事後分布
- `p(u)`
  - 正規化項

## なぜ `p(u|v)` が大事なのか

`p(u|v)` は

「サイズが `v` だったときに、今回の観測 `u` がどれくらい起こりやすいか」

を表します。

正規分布は平均の近くで最も大きく、平均から離れるほど小さくなります。したがって、この例では

- `g(v)` が `u` に近い
  - `p(u|v)` は大きい
- `g(v)` が `u` から遠い
  - `p(u|v)` は小さい

となります。

今回 `g(v)=v^2`、`u=2` なので、

`v^2 ≈ 2`

を満たす `v`、つまり `sqrt(2)` 付近で尤度が高くなります。

## グラフの意味

このファイルが最後に描いているのは

`p(v|u)`

のグラフです。

- 横軸: `v`
- 縦軸: `p(v|u)`

です。

つまり、

「観測 `u=2` が与えられたときに、各サイズ `v` がどれくらいもっともらしいか」

を描いています。

横軸が `v` なのは、`u` を固定して `v` を動かしているからです。

## この例で起きていること

この問題では、2つの力が働いています。

- 事前分布 `p(v)`
  - `v=3` 付近を押す
- 尤度 `p(u|v)`
  - `v≈sqrt(2)` 付近を押す

事後分布 `p(v|u)` は、この2つを掛け合わせた結果です。

したがって、グラフは

- 観測だけの結果
- 事前知識だけの結果

のどちらか一方ではなく、その両方を統合した形になります。

## コードとの最小限の対応

コードでは、数式の各項を次のように計算しています。

- `normpdf(v_range, v_p, sigma_p)`
  - `p(v)`
- `normpdf(u, g(v_range), sigma_u)`
  - `p(u|v)`
- それらの積
  - `p(v)p(u|v)`
- 正規化後の値
  - `p(v|u)`

つまり、このファイルはベイズの定理をそのまま数値的に計算して、最後に `p(v|u)` をグラフにしているだけです。

## 読み方のポイント

このファイルを理解する上で重要なのは、次の順番で考えることです。

1. `p(v)` は「観測前の先入観」
2. `p(u|v)` は「そのサイズが観測をどれだけ説明できるか」
3. `p(v|u)` は「その2つを統合した最終判断」

この3つを分けて考えると、数式とグラフの対応が見えやすくなります。
